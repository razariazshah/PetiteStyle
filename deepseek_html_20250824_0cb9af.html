GitHub Pages Website Guide

Step-by-step instructions to create and deploy your website

1

Create a New Repository

After logging into GitHub, click the "+" icon in the top right corner and select "New repository".

[Screenshot: GitHub new repository button]
The repository name must be in the format: username.github.io (replace username with your GitHub username)
2

Configure Repository Settings

Make sure to set your repository as Public (required for free GitHub Pages).

[Screenshot: Repository visibility options]

Initialize with a README file to make it easier to clone.

3

Upload Your Website Files

Click the "Add file" button and select "Upload files".

[Screenshot: Add file button on GitHub]

Drag and drop your HTML, CSS, and JavaScript files or select them from your computer.

Your main page must be named index.html for it to work properly
4

Enable GitHub Pages

Go to your repository Settings, then scroll down to the "GitHub Pages" section.

[Screenshot: GitHub Pages settings location]

Select the main branch as the source and click Save.

5

Access Your Website

Your website will be available at https://username.github.io within 1-2 minutes.

https://yourusername.github.io
You can also add a custom domain later in the GitHub Pages settings
6

Update Your Website

To make changes, edit your files locally and upload them again, or use GitHub's web editor.

[Screenshot: GitHub file edit interface]

Changes will be live within a few minutes after uploading.

Your website is now live! Share your URL with others to show off your work.
